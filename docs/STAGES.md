# Build Stages

## Stage 0 ‚Äî full first build (this session)

Built the whole deterministic sim + presentation layer in one pass, following the pattern established by the Prologue and Wrong Sky (each phase rebuilds its own determinism spine rather than sharing runtime code).

**Sim (`src/sim/`):** `rng.js`/`canonical.js`/`fingerprint.js` (sfc32 + canonical serialization + FNV-1a golden fingerprint, ported pattern), `visibility.js` (integer Bresenham line-of-sight, used by enemy AI detection), `pathfind.js` (deterministic BFS + a flee-step helper, fixed neighbor order/FIFO frontier/first-write-wins), `ai.js` (the new enemy/car decision system, called from `reduce()`'s `TICK` case ‚Äî see `docs/PROPOSAL.md` ¬ß3 for why it lives there and not in the presentation layer), `content.js`/`world.js`/`reduce.js`/`validate.js`/`demo.js`/`saga.js`/`daynight.js`/`info.js` (the established quest/combat/saga spine, adapted: no wells/visual-facet system this time, `perception` extended with a second AI-legibility threshold, saga imports Wrong Sky's `saga.v2`/exports `saga.v3`).

**Presentation (`src/app/`):** `game.js` (orchestrator ‚Äî audio primed on the player's first real input, never gesturelessly at boot, per the lesson from Wrong Sky's audio bug), `renderer.js` (city tileset: sidewalk/road tiles, procedurally-drawn multi-tile buildings with the walk-behind fade, cars, guard/player sprites, HUD), `audio.js` (simpler than Wrong Sky's ‚Äî no mute/unmute narrative beat needed here, `prime()` creates the context at full volume in one step, still gated on a real gesture), `input.js`/`save.js`/`title.js`/`sprites.js`/`pixelart.js`/`device-labels.js`/`objective-text.js`/`daynight-tint.js` (ported patterns, adapted content).

**Content:** the Lower Banks district ‚Äî a cross-shaped road network, 4 buildings, 2 quest-giver/shop NPCs (Ferro: all quests; Ossa: shop + flavor only ‚Äî see the bug note below), 3 regular guard kinds (Sentry/Bulwark/Cutthroat, the latter with a flee behavior) plus The Warden (boss), 2 ambient cars, a 5-quest chain culminating in a boss fight and a spare/depose fate choice with a real mechanical reward (`depose` grants +1 melee and the `warden-command` technique, carried into `saga.v3`).

**Bug caught before shipping (design-time, not a live incident):** the original content draft gave Ossa both a shop AND two quest-offering duties. `reduce.js`'s `TALK` case fires a `'talked'` event (which `game.js` uses to open a shop modal if the NPC has one) and, separately, a `'quests_offered'` event if any quest's prereqs are met ‚Äî and the `quests_offered` handler unconditionally overwrites `view.modal`. Wrong Sky and the Prologue both avoided this by construction (their shop NPC never gave quests), so the race was never exercised before. Fixed by routing every quest through Ferro and keeping Ossa shop/flavor-only, matching the established pattern instead of patching around the race.

**Bug caught during e2e testing:** the browser end-to-end test initially reused the Node smoke suite's scripted `demoCommands()` walk-to-fixed-coordinates approach against the LIVE running game (via `window.__game.dispatch`) and failed non-deterministically ("quest not offered") ‚Äî because this game's enemies genuinely wander over real time (TICK-driven AI), while the two earlier games' enemies never moved, so their live e2e tests never needed to account for it. Root cause: real wall-clock time (menu navigation, screenshots) passes before the scripted play begins, during which the live render loop's own periodic `TICK` dispatch lets a guard patrol away from the coordinate the test script assumed it would still be at. This is correct game behavior, not a bug ‚Äî fixed the TEST to read enemy positions live and walk toward wherever they currently are (what a real player has to do too), while keeping the pure `reduce()`-only golden-fingerprint check (no real time involved) as the determinism regression test.

**Verified before shipping:** `npm run smoke` (43/43), a browser e2e (boot, zero console errors, golden fingerprint match via a pure `reduce()` replay, an adaptive live playthrough proving the AI/car systems run correctly over real time, and a pixel-sampled confirmation that the building walk-behind fade actually changes rendered color), and a frame-time sanity check (max ~50ms/frame in headless Chromium, no stalls ‚Äî no repeat of Wrong Sky's per-draw-call filter bug, since this renderer never applies `ctx.filter` in a per-tile/per-entity hot loop).

## Stage 0.1 ‚Äî playtest feedback round

Three issues from a real playtest, each researched before fixing:

1. **Aura reverted to a ring instead of the charge-flame effect.** Stage 0 simplified the player's aura visual back to the phase-1 ring instead of carrying forward the DBZ-style charge-flame overlay built for Wrong Sky. Ported the exact effect (`auraFlameAlpha`/`drawAuraFlame`/`drawLick` in `renderer.js`, the `auraFadeActive`/`auraFadeStart`/`auraFadeDuration` view state and charge-release transition logic in `game.js`) ‚Äî full opacity while charging, fading out over 100ms (below 80% aura held) to 500ms (100% aura held) after release.

2. **Enemies spawned instantly on quest accept, with no warning.** Quest-unlocked enemies (e.g. `bulwark-elite1`) used to materialize fully alive in `state.enemies` the instant `ACCEPT_QUEST` was dispatched. Fixed with a genuine sim-side telegraph, not a cosmetic one: `state.pendingSpawns` (an array of `{id, tmpl, readyTick}`) queues the spawn at accept time (emitting an `enemy_incoming` toast), and `reduce()`'s `TICK` case drains any entry whose `readyTick` has arrived ‚Äî `ENEMY_SPAWN_DELAY_TICKS = 3` (1.5s at this game's tick cadence) ‚Äî before that tick's AI decisions run, so a freshly-materialized enemy gets one free beat before it can act. Considered and rejected a presentation-only fade-in: the entity would still be mechanically live and able to strike the player during the "fade," which doesn't actually address "shouldn't spawn instantly" (an attackability complaint, not a sprite-pop complaint). The boss's existing instant-spawn-then-modal pattern was confirmed fine as-is and left untouched ‚Äî `handleWorld()` (and therefore all TICK/MOVE/ENEMY_STRIKE dispatch) is gated behind `!view.modal`, so the boss is provably inert until the player dismisses the appearance dialog; the modal itself already is the telegraph.

3. **A "kill" quest could be permanently soft-locked by killing its target before accepting it.** `read-the-city` (kill 1 sentry) and `watch-them-move` (kill 1 cutthroat) pointed at `sentry1`/`cutthroat1` ‚Äî free-roam enemies present from world start, unlike the finale quest's correctly existence-gated `bulwark-elite1`. A player who killed either before ever accepting the corresponding quest permanently soft-locked it (and everything chained after it via `requires`) with no error, no crash, no player-visible signal ‚Äî just a quest that silently never completes. A full audit of every quest turned up a THIRD instance of the same bug: `ferros-ledger`'s `signet` collect-objective pointed at a free pickup with the identical "gone before you needed it" risk. Fixed by existence-gating all three (`unlocks.enemies`/`unlocks.pickups` on their respective quests), mirroring the one correct precedent instead of inventing a new mechanism. Required updating `demo.js` (insert 3 `TICK`s after each affected `ACCEPT_QUEST` so the newly-gated enemy actually appears before the scripted fight) and `scripts/smoke.mjs` (several AI/immunity unit tests that referenced `sentry1`/`cutthroat1` off a bare `makeWorld()` now build a synthetic test enemy instead, mirroring the pattern the `bulwark` immunity test already used).

All three fixes researched via parallel investigation passes before any code changed (a full quest-by-quest audit for the soft-lock pattern, a design pass on deterministic spawn-telegraph mechanisms respecting the `src/sim` determinism guard, and an exact reconstruction of Wrong Sky's aura-flame code for a verbatim port) ‚Äî re-verified with the full smoke suite (45/45, new golden fingerprint `d099c9ea`) and a browser e2e covering all three fixes live (spawn-telegraph timing, the fixed soft-lock, and the flame rendering correctly in a screenshot).
